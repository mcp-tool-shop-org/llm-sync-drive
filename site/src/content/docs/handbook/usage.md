## CLI commands

All commands accept `-v` / `--verbose` for debug logging.

### `llm-sync-drive init`

Create a default `llm-sync-drive.yaml` config file in the current directory.

```bash
llm-sync-drive init --repo .
```

### `llm-sync-drive auth`

Authenticate with Google Drive (OAuth mode only). Opens a browser for consent and caches the token.

```bash
llm-sync-drive auth
```

ADC and service account modes don't need this step.

### `llm-sync-drive sync`

One-shot compile and upload. Compiles the repo into structured Markdown, then uploads to Google Drive.

```bash
llm-sync-drive sync
```

### `llm-sync-drive serve`

Watch mode with auto-sync. Runs an initial sync, then watches the repo for file changes. After the debounce interval (default 5 seconds), it recompiles and uploads.

```bash
llm-sync-drive serve
```

Press `Ctrl+C` to stop watching.

### `llm-sync-drive compile`

Compile locally without uploading. Useful for inspecting the output before syncing.

```bash
llm-sync-drive compile -o llms.txt
```

## Output format

The compiled `llms.txt` follows this structure:

```markdown
# project-name

> Auto-generated by llm-sync-drive on 2026-03-12 18:00 UTC
> Source: /path/to/repo
> Files included: 42

## Directory Structure

src/
  index.ts
  utils/
    helpers.ts

## File Contents

### src/index.ts

// file contents here
```

Each file gets a section header (`### path/to/file`) with the contents in a fenced code block tagged with the file extension for syntax highlighting.

## Ignore rules

Files are excluded in this order of priority:

1. **Built-in rules** — `.git/`, `node_modules/`, `__pycache__/`, lock files, secret files (`.env`, `*.key`, `*.pem`)
2. **`.gitignore`** — standard git ignore patterns
3. **`.llmsignore`** — additional patterns specific to LLM context (e.g., test fixtures, binary assets)
4. **`extra_ignore_patterns`** in config — per-repo overrides

See `.llmsignore.example` in the repo for a starter template.
